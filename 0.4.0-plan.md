## Item 1: Stackful Coroutines (P0)

### Problem

Current coroutines are pthread-based: each `dispatch` block spawns an OS thread with a full 8MB stack, synchronizes with mutex+condvar for each yield. This means:
- Thread creation: ~50μs overhead per coroutine
- Memory: 8MB per OS stack (Linux default), 160 bytes for sync struct = ~8MB total per coroutine
- Max coroutines: ~few thousand before OOM or thread exhaustion
- Yield cost: 2 mutex lock/unlock + condvar wait/signal = ~2-5μs per yield

### Target

- Coroutine creation: ~50ns (malloc 8KB + write 3 pointers)
- Memory: 8KB initial stack, growable to 1MB on demand
- Max coroutines: millions (limited only by heap)
- Yield cost: ~5ns (save registers + swap stack pointer = ~15 instructions)
- Context switch: setjmp/longjmp on portable path, raw asm for x86_64/aarch64 fast path

### Design: Context Switch Mechanism

Use a **minimal platform-specific context struct** that saves only callee-saved registers (the ABI guarantees caller-saved registers are already handled by the compiler at call sites). This is what Go, Lua, and libco do internally.

#### x86_64 Context (56 bytes)

```c
typedef struct {
    void *rsp;   // stack pointer
    void *rbp;   // frame pointer
    void *rbx;   // callee-saved
    void *r12;   // callee-saved
    void *r13;   // callee-saved
    void *r14;   // callee-saved
    void *r15;   // callee-saved
} jade_context_t;     // 7 × 8 = 56 bytes
```

#### aarch64 Context (160 bytes)

```c
typedef struct {
    void *sp;          // stack pointer
    void *lr;          // link register (x30)
    void *fp;          // frame pointer (x29)
    void *x19_x28[10]; // callee-saved x19–x28
    double d8_d15[8];  // callee-saved SIMD/FP d8–d15
} jade_context_t;      // 20 × 8 = 160 bytes
```

#### Context Switch (x86_64, ~15 instructions)

```asm
; jade_context_swap(jade_context_t *from, jade_context_t *to)
; Save current context into *from, restore *to
jade_context_swap:
    ; Save callee-saved registers into [rdi] (from)
    mov [rdi +  0], rsp
    mov [rdi +  8], rbp
    mov [rdi + 16], rbx
    mov [rdi + 24], r12
    mov [rdi + 32], r13
    mov [rdi + 40], r14
    mov [rdi + 48], r15

    ; Restore callee-saved registers from [rsi] (to)
    mov rsp, [rsi +  0]
    mov rbp, [rsi +  8]
    mov rbx, [rsi + 16]
    mov r12, [rsi + 24]
    mov r13, [rsi + 32]
    mov r14, [rsi + 40]
    mov r15, [rsi + 48]

    ret   ; returns to the address on the new stack
```

This is exactly what Go's `runtime.gogo` does (look at `src/runtime/asm_amd64.s` in the Go source). The `ret` instruction pops the return address from the new stack, resuming where the coroutine last yielded.

#### Portable Fallback (setjmp/longjmp)

```c
#include <setjmp.h>

typedef struct {
    jmp_buf env;
} jade_context_t;

static inline void jade_context_swap(jade_context_t *from, jade_context_t *to) {
    if (setjmp(from->env) == 0) {
        longjmp(to->env, 1);
    }
}
```

Drawback: setjmp/longjmp save/restore signal masks on some platforms (glibc), adding ~200ns. Use `_setjmp`/`_longjmp` (BSD/Linux) or `sigsetjmp` with `savesigs=0` to avoid this. Still 10-50× slower than raw asm.

**Decision:** Ship with x86_64 asm as default (it's our primary target), aarch64 asm for ARM, `_setjmp`/`_longjmp` as fallback. Detection at build time via `#ifdef __x86_64__` / `#ifdef __aarch64__`.

### Design: Stack Management

#### Stack Layout

```
            ┌──────────────────┐  ← stack_base + stack_size (high address)
            │   guard page     │  4KB, PROT_NONE (segfault on overflow)
            │   (mprotect)     │
            ├──────────────────┤
            │                  │
            │   usable stack   │  8KB - 4KB guard = 4KB usable initially
            │   (grows down)   │
            │                  │
            ├──────────────────┤  ← rsp (current stack pointer)
            │                  │
            │   unused space   │
            │                  │
            └──────────────────┘  ← stack_base (low address, mmap'd)
```

#### Initial Allocation

```c
#define JADE_STACK_INIT   (8 * 1024)   // 8KB initial
#define JADE_STACK_MAX    (1024 * 1024) // 1MB max
#define JADE_GUARD_SIZE   (4096)        // one page guard

void *jade_stack_alloc(size_t size) {
    // mmap anonymous memory, aligned to page boundary
    void *base = mmap(NULL, size, PROT_READ | PROT_WRITE,
                      MAP_PRIVATE | MAP_ANONYMOUS | MAP_STACK, -1, 0);
    if (base == MAP_FAILED) return NULL;

    // Guard page at the bottom (stack grows down)
    mprotect(base, JADE_GUARD_SIZE, PROT_NONE);

    return base;
}
```

#### Growable Stacks (Split/Copy Strategy)

Go uses **stack copying** (contiguous stacks): when the stack probe fails, allocate a 2× larger stack, copy the old stack content, update all pointers. This is excellent for Go because the Go compiler knows every pointer on the stack (precise GC).

Jade doesn't have precise stack maps. Instead, use the **segmented stack** approach with a twist:

1. **Compiler inserts stack probes** at function prologues (same as `__morestack` in GCC's split-stack mode)
2. If `rsp - frame_size < stack_guard`, call `jade_stack_grow()`
3. `jade_stack_grow()` does `mremap()` (Linux) to extend the mapping in-place, or allocates a new larger stack and copies (with `memcpy`, no pointer fixup needed since we don't store interior stack pointers)

For simplicity and correctness in v1: **do not grow**. 8KB is the stack. If the user stack-overflows, they hit the guard page and get a segfault. This is identical to how Go 1.0 worked before contiguous stacks. Growable stacks can be added later per the roadmap.

In practice, 8KB is sufficient for the vast majority of coroutine/actor workloads (message handlers, channel consumers, pipeline stages). Deep recursion inside a coroutine is an anti-pattern anyway — it should be a regular function.

**v1 decision:** Fixed 8KB stacks with guard page. Segfault on overflow (detected and reported via signal handler). No growing.

### Design: Coroutine Object

```c
typedef enum {
    JADE_CORO_READY,     // Created, never run
    JADE_CORO_RUNNING,   // Currently executing on a worker
    JADE_CORO_SUSPENDED, // Yielded or blocked on channel
    JADE_CORO_DONE       // Completed, awaiting cleanup
} jade_coro_state_t;

typedef struct jade_coro {
    jade_context_t    ctx;           // saved registers (56 bytes on x86_64)
    void             *stack_base;    // mmap'd stack memory
    uint32_t          stack_size;    // current stack size
    jade_coro_state_t state;         // lifecycle state
    void            (*entry)(void*); // coroutine body function pointer
    void             *arg;           // argument to entry (closure env / actor state)
    struct jade_coro *next;          // intrusive linked list (for run queue / wait list)
    void             *wait_chan;     // channel this coro is blocked on (NULL if not)
    uint32_t          id;            // unique id (for debugging)
} jade_coro_t;  // ~112 bytes on x86_64 + 8KB stack = ~8.3KB total
```

```

                              │
                              ▼
                        ┌──────────┐
                        │  READY   │
                        └────┬─────┘
                             │ scheduled onto worker run queue
                             ▼
                        ┌──────────┐  yield / channel block
             resume ──► │ RUNNING  │ ──────────────────────►┌───────────┐
                        └──────────┘                        │ SUSPENDED │
```

#### 1c. Create `runtime/coro.c` — coroutine create/destroy

```c
jade_coro_t *jade_coro_create(void (*entry)(void*), void *arg) {
    jade_coro_t *c = calloc(1, sizeof(jade_coro_t));
    c->stack_base = jade_stack_alloc(JADE_STACK_INIT);
    c->stack_size = JADE_STACK_INIT;
    c->entry = entry;
    c->arg = arg;
    c->state = JADE_CORO_READY;
    c->id = atomic_fetch_add(&g_coro_id_counter, 1);

    // Set up initial stack frame:
    // The stack must look like entry() was called with arg as parameter
    void *stack_top = (char*)c->stack_base + c->stack_size;
    // Align to 16 bytes (ABI requirement)
    stack_top = (void*)((uintptr_t)stack_top & ~15ULL);
    // Push a fake return address that points to jade_coro_exit
    stack_top = (char*)stack_top - 8;
    *(void**)stack_top = (void*)jade_coro_exit;
    // Push the entry point as the "return address" for context_swap's ret
    stack_top = (char*)stack_top - 8;
    *(void**)stack_top = (void*)jade_coro_trampoline;

    c->ctx.rsp = stack_top;
    c->ctx.rbp = stack_top;
    c->ctx.r12 = (void*)entry;  // stash entry in r12 (callee-saved)
    c->ctx.r13 = arg;            // stash arg in r13 (callee-saved)

    return c;
}

// Trampoline: called the first time the coroutine is swapped to
static void jade_coro_trampoline(void) {
    jade_coro_t *self = jade_current_coro();
    // r12 has entry, r13 has arg (set during create)
    void (*entry)(void*);
    void *arg;
    __asm__ volatile ("mov %%r12, %0" : "=r"(entry));
    __asm__ volatile ("mov %%r13, %0" : "=r"(arg));
    entry(arg);
    jade_coro_exit();  // never returns
}

static void jade_coro_exit(void) {
    jade_coro_t *self = jade_current_coro();
    self->state = JADE_CORO_DONE;
    jade_sched_yield_final();  // return to scheduler, never resume
    __builtin_unreachable();
}
```

## Item 2: Typed Channels (P0)

### Problem

There is no first-class channel type. The current coroutine system uses a single i64 value slot with mutex+condvar synchronization — this loses type information (`coerce_to_i64` truncates floats, can't pass structs) and can only buffer one value at a time.

### Target

```jade
ch is channel of i64           # bounded channel, default capacity 64
ch is channel of String, 1024  # bounded, capacity 1024

send ch, 42                    # blocks if full
val is receive ch              # blocks if empty

close ch                       # signal no more values
```

Channels are the composition primitive. They replace the current coroutine value slot and can be used independently from coroutines.

### Design: Channel Layout

A channel is a **bounded MPMC ring buffer** with wait lists for blocked senders/receivers. This is almost identical to the current actor mailbox, except:
1. Elements are typed (not tag+payload)
2. Multiple producers AND multiple consumers (MPMC, not MPSC)
3. Close semantics (recv returns `None` after close)
4. Integration with the scheduler's coroutine suspend/resume (not OS condvar)

#### Runtime Channel Struct

```c
typedef struct jade_chan {
    _Atomic(uint64_t) head;           // consumer position
    _Atomic(uint64_t) tail;           // producer position
    uint64_t          capacity;       // ring buffer size (power of 2)
    size_t            elem_size;      // sizeof(T) in bytes
    void             *buffer;         // ring buffer: capacity × elem_size bytes
    _Atomic(int32_t)  closed;         // 1 = closed, no more sends allowed
    _Atomic(int32_t)  sender_count;   // number of blocked senders
    _Atomic(int32_t)  recver_count;   // number of blocked receivers
    jade_coro_t      *send_waitq;     // linked list of blocked senders
    jade_coro_t      *recv_waitq;     // linked list of blocked receivers
    pthread_mutex_t   waitq_lock;     // protects wait queues (NOT the fast path)
} jade_chan_t;
```

#### Why Not Lock-Free for the Channel Itself?

Lock-free MPMC queues (e.g., Dmitry Vyukov's bounded MPMC) are excellent for raw throughput but don't support **blocking with coroutine suspend**. The core operation is:

1. Try to enqueue → if full, **suspend the current coroutine** onto the channel's wait queue
2. When a dequeue happens → **resume one suspended sender**

Suspending a coroutine requires manipulating its state and the wait queue, which needs synchronization. The fast path (buffer not full/not empty) doesn't need to touch the wait queue at all.

**Hybrid approach** (what Go does):

```
Send fast path (no lock):
  1. Atomic load tail, head
  2. If (tail - head) < capacity → CAS tail → memcpy elem → return
  3. Else → fall through to slow path

Send slow path (locked):
  1. Lock waitq_lock
  2. Double-check: still full?
  3. If a receiver is waiting on recv_waitq → dequeue receiver, hand off element directly, resume receiver
  4. Else → park current coroutine on send_waitq, unlock, yield to scheduler
```

This gives:
- **Uncontended send/recv: ~10ns** (atomic CAS + memcpy)
- **Contended send/recv: ~50ns** (lock + wake)
- **Full/empty: ~5ns** (yield to scheduler, no OS call)

#### Direct Handoff Optimization

When a sender finds a waiting receiver (or vice versa), skip the buffer entirely and copy the element directly from sender's memory to receiver's memory. This is Go's fast path and eliminates one buffer copy.

```c
void jade_chan_send(jade_chan_t *ch, const void *data) {
    // Fast path: try to write into ring buffer
    uint64_t tail = atomic_load_explicit(&ch->tail, memory_order_relaxed);
    uint64_t head = atomic_load_explicit(&ch->head, memory_order_acquire);
    if (tail - head < ch->capacity) {
        if (atomic_compare_exchange_weak(&ch->tail, &tail, tail + 1)) {
            size_t idx = tail & (ch->capacity - 1);  // bitmask modulo
            memcpy((char*)ch->buffer + idx * ch->elem_size, data, ch->elem_size);
            // Check if a receiver is waiting
            if (atomic_load(&ch->recver_count) > 0) {
                jade_chan_wake_one_recver(ch);
            }
            return;
        }
        // CAS failed, retry
        goto retry;
    }

    // Slow path: buffer full, park on send_waitq
    jade_chan_send_slow(ch, data);
}
```

### Design: Bridging Coroutines and Channels

The `dispatch` block currently yields values through a single-slot mutex+condvar pipe. Replace this with a channel:

## Item 3: M:N Scheduler (P1)

### Problem

Current model: 1 OS thread per actor/coroutine. With items 1 and 2, coroutines are cheap and channels are efficient, but there's no scheduler to multiplex M coroutines onto N OS threads. Without a scheduler, `jade_coro_yield()` has nowhere to yield TO.

### Target

- N worker threads (default = `num_cpus`, configurable)
- M coroutines scheduled across workers
- Work-stealing for load balance
- Coroutines that block on channel ops are parked off the run queue
- Total overhead: ~50ns per schedule decision

### Design: Work-Stealing Deque

Each worker thread has a **local deque** (double-ended queue). The owning worker pushes/pops from the bottom (LIFO — cache-friendly, locality). Idle workers steal from the top (FIFO — coarse-grained, amortized).

This is the Chase-Lev deque, used by Cilk, Go, Tokio, and Rayon.

#### Chase-Lev Deque Implementation

```c
#define INITIAL_DEQUE_SIZE 256

typedef struct {
    _Atomic(jade_coro_t*) *buffer;  // circular buffer of coro pointers
    _Atomic(int64_t) top;            // steal end (other workers CAS here)
    _Atomic(int64_t) bottom;         // push/pop end (owning worker only)
    _Atomic(int64_t) capacity;       // power of 2
} jade_deque_t;

void jade_deque_push(jade_deque_t *dq, jade_coro_t *c) {
    int64_t b = atomic_load_explicit(&dq->bottom, memory_order_relaxed);
    int64_t t = atomic_load_explicit(&dq->top, memory_order_acquire);
    if (b - t >= atomic_load(&dq->capacity)) {
        jade_deque_grow(dq);  // double the buffer
    }
    int64_t cap = atomic_load(&dq->capacity);
    atomic_store_explicit(&dq->buffer[b & (cap - 1)], c, memory_order_relaxed);
    atomic_thread_fence(memory_order_release);
    atomic_store_explicit(&dq->bottom, b + 1, memory_order_relaxed);
}

jade_coro_t *jade_deque_pop(jade_deque_t *dq) {
    int64_t b = atomic_load_explicit(&dq->bottom, memory_order_relaxed) - 1;
    atomic_store_explicit(&dq->bottom, b, memory_order_relaxed);
    atomic_thread_fence(memory_order_seq_cst);
    int64_t t = atomic_load_explicit(&dq->top, memory_order_relaxed);
    if (t <= b) {
        int64_t cap = atomic_load(&dq->capacity);
        jade_coro_t *c = atomic_load_explicit(&dq->buffer[b & (cap - 1)], memory_order_relaxed);
        if (t == b) {
            // Last element — race with steal
            if (!atomic_compare_exchange_strong(&dq->top, &t, t + 1)) {
                c = NULL; // stolen
            }
            atomic_store_explicit(&dq->bottom, b + 1, memory_order_relaxed);
        }
        return c;
    }
    // Empty
    atomic_store_explicit(&dq->bottom, b + 1, memory_order_relaxed);
    return NULL;
}

jade_coro_t *jade_deque_steal(jade_deque_t *dq) {
    int64_t t = atomic_load_explicit(&dq->top, memory_order_acquire);
    atomic_thread_fence(memory_order_seq_cst);
    int64_t b = atomic_load_explicit(&dq->bottom, memory_order_acquire);
    if (t < b) {
        int64_t cap = atomic_load(&dq->capacity);
        jade_coro_t *c = atomic_load_explicit(&dq->buffer[t & (cap - 1)], memory_order_relaxed);
        if (atomic_compare_exchange_strong_explicit(&dq->top, &t, t + 1,
                memory_order_seq_cst, memory_order_relaxed)) {
            return c;
        }
    }
    return NULL; // empty or contended
}
```

### Design: Scheduler

```c

    int             num_workers;     // N
    _Atomic(int64_t) active_coros;   // total non-done coroutines
    _Atomic(int32_t) shutdown;       // 1 = shutting down

    // Global inject queue for cross-worker spawns
    jade_coro_t    *inject_head;
    jade_coro_t    *inject_tail;
    pthread_mutex_t inject_lock;

    pthread_mutex_t idle_lock;
    pthread_cond_t  idle_cond;
```

### Design: Worker Loop

```c
static void *jade_worker_loop(void *arg) {
    jade_worker_t *w = (jade_worker_t*)arg;
    tl_worker = w;

    while (!atomic_load(&g_sched.shutdown)) {
        jade_coro_t *c = jade_find_work(w);
        if (c == NULL) {
            // No work anywhere — park this thread
            jade_worker_park(w);
            continue;
        }

        // Run the coroutine
        c->state = JADE_CORO_RUNNING;
        w->current = c;
        jade_context_swap(&w->sched_ctx, &c->ctx);
        // Coroutine yielded or completed — back in scheduler
        w->current = NULL;

        if (c->state == JADE_CORO_DONE) {
            atomic_fetch_sub(&g_sched.active_coros, 1);
            jade_coro_destroy(c);
        } else if (c->state == JADE_CORO_SUSPENDED) {
            // Coroutine is parked on a channel wait queue
            // Don't re-enqueue; the channel will resume it
        } else {
            // READY = yielded voluntarily, put back on run queue
            jade_deque_push(&w->run_queue, c);
        }
    }
    return NULL;
}
```

### Design: Finding Work (Steal Order)

```c
static jade_coro_t *jade_find_work(jade_worker_t *w) {
    // 1. Pop from own deque (LIFO — hot cache)
    jade_coro_t *c = jade_deque_pop(&w->run_queue);
    if (c) return c;

    // 2. Check global inject queue
    c = jade_inject_pop();
    if (c) return c;

    // 3. Steal from random other worker (FIFO)
    int n = g_sched.num_workers;
    uint32_t start = jade_xorshift(&w->rng_state) % n;
    for (int i = 0; i < n; i++) {
        uint32_t victim = (start + i) % n;
        if (victim == w->id) continue;
        c = jade_deque_steal(&g_sched.workers[victim].run_queue);
        if (c) return c;
    }

    return NULL; // no work anywhere
}
```

### Design: Yield

```c
void jade_coro_yield(void) {
    jade_worker_t *w = tl_worker;
    jade_coro_t *c = w->current;
    c->state = JADE_CORO_READY;  // will be re-enqueued by worker loop
    jade_context_swap(&c->ctx, &w->sched_ctx);
    // Resumed here when re-scheduled
}
```

That's 5 lines. The context_swap saves registers, switches stack to the worker's scheduler stack, and the worker loop puts the coroutine back on the deque. When the worker picks it back up, context_swap returns and execution resumes here.

### Design: Channel ↔ Scheduler Integration

When a coroutine blocks on a channel:

```c
// Inside jade_chan_send_slow() or jade_chan_recv_slow():
void jade_chan_park_sender(jade_chan_t *ch) {
    jade_worker_t *w = tl_worker;
    jade_coro_t *c = w->current;

    pthread_mutex_lock(&ch->waitq_lock);
    c->state = JADE_CORO_SUSPENDED;
    c->wait_chan = ch;
    c->next = NULL;
    // Append to send wait queue
    if (ch->send_waitq == NULL) {
        ch->send_waitq = c;
    } else {
        jade_coro_t *tail = ch->send_waitq;
        while (tail->next) tail = tail->next;
        tail->next = c;
    }
    atomic_fetch_add(&ch->sender_count, 1);
    pthread_mutex_unlock(&ch->waitq_lock);

    // Yield to scheduler (don't re-enqueue — we're SUSPENDED)
    jade_context_swap(&c->ctx, &w->sched_ctx);
    // Resumed here when channel has space
}

// When a receiver frees space:
void jade_chan_wake_one_sender(jade_chan_t *ch) {
    pthread_mutex_lock(&ch->waitq_lock);
    jade_coro_t *c = ch->send_waitq;
    if (c) {
        ch->send_waitq = c->next;
        c->next = NULL;
        c->wait_chan = NULL;
        atomic_fetch_sub(&ch->sender_count, 1);
    }
    pthread_mutex_unlock(&ch->waitq_lock);

    if (c) {
        c->state = JADE_CORO_READY;
        // Push onto current worker's run queue (locality) or inject queue
        jade_sched_enqueue(c);
    }
}
```

### Design: Worker Parking and Wakeup

When a worker finds no work, it parks using pthread condvar:

```c
static void jade_worker_park(jade_worker_t *w) {
    atomic_fetch_add(&g_sched.idle_count, 1);
    pthread_mutex_lock(&g_sched.idle_lock);
    // Double-check for work before sleeping (avoid lost wakeups)
    if (jade_deque_pop(&w->run_queue) == NULL && !atomic_load(&g_sched.shutdown)) {
        pthread_cond_wait(&g_sched.idle_cond, &g_sched.idle_lock);
    }
    pthread_mutex_unlock(&g_sched.idle_lock);
    atomic_fetch_sub(&g_sched.idle_count, 1);
}

static void jade_sched_wake_one(void) {
    if (atomic_load(&g_sched.idle_count) > 0) {
        pthread_cond_signal(&g_sched.idle_cond);
    }
}
```

### Design: Scheduler Init and Shutdown

```c
void jade_sched_init(int num_workers) {
    if (num_workers <= 0) {
        num_workers = sysconf(_SC_NPROCESSORS_ONLN);
        if (num_workers <= 0) num_workers = 4;
    }
    g_sched.num_workers = num_workers;
    g_sched.workers = calloc(num_workers, sizeof(jade_worker_t));
    atomic_store(&g_sched.active_coros, 0);
    atomic_store(&g_sched.shutdown, 0);
    pthread_mutex_init(&g_sched.inject_lock, NULL);
    pthread_mutex_init(&g_sched.idle_lock, NULL);
    pthread_cond_init(&g_sched.idle_cond, NULL);

    for (int i = 0; i < num_workers; i++) {
        g_sched.workers[i].id = i;
        g_sched.workers[i].rng_state = i + 1; // nonzero seed
        jade_deque_init(&g_sched.workers[i].run_queue);
        // Don't start threads yet — start on first spawn
    }
}

void jade_sched_spawn(jade_coro_t *c) {
    atomic_fetch_add(&g_sched.active_coros, 1);

    // Start worker threads lazily on first spawn
    static _Atomic(int) started = 0;
    if (atomic_exchange(&started, 1) == 0) {
        for (int i = 0; i < g_sched.num_workers; i++) {
            pthread_create(&g_sched.workers[i].thread, NULL,
                           jade_worker_loop, &g_sched.workers[i]);
        }
    }

    // Inject into global queue (or local if called from a worker)
    jade_worker_t *w = tl_worker;
    if (w) {
        jade_deque_push(&w->run_queue, c);
    } else {
        jade_inject_push(c);
    }
    jade_sched_wake_one();
}

void jade_sched_run(void) {
    // Block until all coroutines complete
    while (atomic_load(&g_sched.active_coros) > 0) {
        sched_yield();  // or futex wait with periodic check
    }
}

void jade_sched_shutdown(void) {
    atomic_store(&g_sched.shutdown, 1);
    // Wake all idle workers
    pthread_cond_broadcast(&g_sched.idle_cond);
    for (int i = 0; i < g_sched.num_workers; i++) {
        pthread_join(g_sched.workers[i].thread, NULL);
    }
    free(g_sched.workers);
}
```

## Item 4: Migrate Actors to Scheduler (P1)

### Problem

Actors currently use `pthread_create` + `pthread_detach` per actor. This means:
- Each actor consumes one OS thread (~8MB stack)
- Max actors limited by thread count (~10K)
- The `usleep()` hack at end of main to wait for actors to finish
- Actor threads can't be reclaimed when actors are idle

### Target

Same actor syntax, same semantics, 1000× less overhead:
- Each actor becomes a coroutine on the scheduler
- Actor mailbox remains (MPSC ring buffer) but uses scheduler-aware parking instead of condvar
- `spawn Actor` creates a coroutine, not a thread
- `send actor, @msg()` wakes the actor coroutine if it's parked on empty mailbox
- No `usleep()` needed — `jade_sched_run()` blocks until all actors complete

### Design: Actor as Coroutine

The actor loop function (`{Name}_loop`) currently takes a mailbox pointer, runs `{ dequeue → switch → handle }` in an infinite loop on its own pthread. Transform it to:

```c
// Before (current):
void *Counter_loop(void *arg) {
    Counter_mailbox *mb = arg;
    while (mb->alive) {
        pthread_mutex_lock(&mb->mutex);
        while (mb->count == 0 && mb->alive)
            pthread_cond_wait(&mb->cond_ne, &mb->mutex);
        if (!mb->alive) { pthread_mutex_unlock(&mb->mutex); break; }
        // dequeue + dispatch
        pthread_mutex_unlock(&mb->mutex);
    }
    return NULL;
}

// After (coroutine):
void Counter_loop(void *arg) {
    Counter_mailbox *mb = arg;
    while (mb->alive) {
        while (mb->count == 0 && mb->alive) {
            // Instead of condvar wait: park coroutine
            jade_actor_park(mb);  // sets state=SUSPENDED, yields to scheduler
        }
        // dequeue + dispatch (same as before, no locks needed on single-consumer side)
        jade_coro_yield();  // yield after each message to be fair
    }
}
```

#### Mailbox Changes

The mailbox currently uses mutex+condvar for synchronization. With the scheduler:

**Enqueue side (send, called from any coroutine/thread):**
- Still needs lock or atomics (multiple producers)
- After successful enqueue, if the actor coroutine is SUSPENDED on this mailbox, wake it: `jade_sched_enqueue(actor_coro)`
- No condvar signal needed

**Dequeue side (actor loop, single consumer):**
- No lock needed (single consumer invariant)
- If mailbox empty, park the coroutine on the mailbox's wait slot
- When resumed, dequeue and dispatch

The **lock on the producer side** can be eliminated in favor of a lock-free MPSC queue. Since the actor is the sole consumer and multiple senders are producers, an atomic MPSC queue is ideal:

```c
typedef struct jade_actor_mailbox {
    // Lock-free MPSC ring buffer (same as current layout but atomic head/tail)
    void            *buffer;        // message array
    _Atomic(uint64_t) head;         // consumer reads from here
    _Atomic(uint64_t) tail;         // producers CAS here
    uint64_t         capacity;      // power of 2
    uint64_t         msg_size;      // bytes per message
    _Atomic(int32_t) alive;         // 1 = running
    jade_coro_t     *actor_coro;    // the coroutine running this actor
    void            *state;         // pointer to actor state struct
} jade_actor_mailbox_t;
```

### Design: Codegen Changes

#### compile_spawn

Replace:

```

```

With:


#### compile_send

```

```

With:
```

```

The send operation becomes lock-free on the fast path. The `compile_send` function drops from ~150 lines to ~60 lines.

#### compile_actor_loop

The actor loop is still compiled as a function that the scheduler calls. The key change is replacing `pthread_cond_wait` with `jade_actor_park`.

Currently the loop codegen is ~230 lines of complex LLVM IR building (mutex lock, condvar wait branching, switch on tag, handler dispatch). Most of this can be simplified:

- Remove all `pthread_mutex_lock/unlock` calls from the loop
- Remove all `pthread_cond_wait` calls
- Replace the "wait for message" spin with a call to `jade_actor_park(mb_ptr)` when the mailbox is empty
- The switch-on-tag and handler dispatch remain exactly the same

Net codegen change: ~230 lines → ~120 lines (remove mutex/condvar scaffolding).

### Design: Actor Stop

With the scheduler, we can wire a clean stop mechanism:

```jade
*main()
    c is spawn Counter
    send c, @inc(1)
    stop c              # new keyword: sets alive=0, wakes actor, reclaims coroutine
```

Codegen: `stop c` emits:
```

    jade_sched_enqueue(mailbox.actor_coro)  // wake it so it can exit
```

The actor loop's `while (alive)` check causes it to exit, marking the coroutine as DONE.

### Design: Actor Reply (Request-Response)

With channels available, request-response becomes trivial:

## Item 5: Select (P2)

### Problem

When a coroutine needs to wait on multiple channels simultaneously (or a channel and a timer, or multiple actor responses), there's no way to do it without polling.

### Target

```jade
select
    val from ch1
        log('got from ch1:', val)
    val from ch2
        log('got from ch2:', val)
    timeout 1000
        log('timed out')
```

Compiles to a multi-channel wait on the scheduler. The coroutine suspends until ANY channel is ready, then executes the corresponding arm.

### Design: Select Semantics

Go's select statement is the gold standard. Key semantics:
1. If multiple channels are ready, choose **randomly** (fairness)
2. If no channel is ready, **block** (park the coroutine)
3. An optional `default` case makes it non-blocking (poll)
4. A channel can be nil to skip that case

For Jade, we follow Go's model but use Jade's syntax:

```jade
select
    val from ch1          # recv from ch1 into val
        handle(val)
    from ch2              # recv from ch2, discard value
        log('ch2 ready')
    send ch3, data        # send data to ch3
        log('sent to ch3')
    timeout 1000          # timeout in ms (optional)
        log('timed out')
    default               # non-blocking, don't mix with timeout
        log('nothing ready')
```

### Design: Runtime Implementation

The key challenge: a single coroutine must register itself on **multiple** channel wait queues simultaneously, then be woken when **any** one fires.

#### Select Case Structure

```c
typedef struct jade_select_case {
    jade_chan_t     *chan;        // channel
    void           *data;        // buffer (recv: output, send: input)
    uint16_t        is_send;     // 0=recv, 1=send
    uint16_t        selected;    // set to 1 if this case was chosen
} jade_select_case_t;
```

#### Select Algorithm (Go's Algorithm)

Go's `runtime.selectgo` is one of the most carefully designed algorithms in systems programming. The algorithm:

1. **Shuffle** the cases (random permutation of indices) for fairness
2. **Lock all channels** in address order (avoid deadlock — total ordering)
3. **Scan** cases in shuffled order: if any channel is ready → execute it, unlock all, return
4. If none ready and `default` exists → unlock all, return default
5. **Enqueue** the coroutine on ALL channel wait queues (it appears as a waiter on every channel)
6. **Unlock all channels**
7. **Park** the coroutine (yield to scheduler)
8. When woken: one channel fired → **lock all channels again**
9. **Dequeue** the coroutine from ALL other channel wait queues (it must only run once)
10. **Unlock all**, return the case that fired

The double-lock in address order is critical to prevent deadlocks when multiple selects overlap.

```c
int jade_select(jade_select_case_t *cases, int n, int has_default) {
    // 1. Create shuffled index order
    uint16_t poll_order[n];
    for (int i = 0; i < n; i++) poll_order[i] = i;
    jade_shuffle(poll_order, n);

    // 2. Create lock order (by channel address, ascending)
    uint16_t lock_order[n];
    for (int i = 0; i < n; i++) lock_order[i] = i;
    jade_sort_by_chan_addr(lock_order, cases, n);

    // 3. Lock all channels
    jade_select_lock_all(cases, lock_order, n);

    // 4. Scan for ready channel
    for (int i = 0; i < n; i++) {
        int idx = poll_order[i];
        jade_select_case_t *c = &cases[idx];
        if (c->is_send) {
            if (jade_chan_can_send(c->chan)) {
                jade_chan_send_locked(c->chan, c->data);
                jade_select_unlock_all(cases, lock_order, n);
                return idx;
            }
        } else {
            if (jade_chan_can_recv(c->chan)) {
                jade_chan_recv_locked(c->chan, c->data);
                jade_select_unlock_all(cases, lock_order, n);
                return idx;
            }
        }
    }

    // 5. Nothing ready — if default, return it
    if (has_default) {
        jade_select_unlock_all(cases, lock_order, n);
        return -1; // default
    }

    // 6. Enqueue on all channel wait queues
    jade_coro_t *self = jade_current_coro();
    jade_select_enqueue_all(self, cases, n);

    // 7. Unlock all and park
    jade_select_unlock_all(cases, lock_order, n);
    self->state = JADE_CORO_SUSPENDED;
    jade_context_swap(&self->ctx, &tl_worker->sched_ctx);

    // 8. Woken up — figure out which case fired
    jade_select_lock_all(cases, lock_order, n);
    int result = jade_select_dequeue_all(self, cases, n);
    jade_select_unlock_all(cases, lock_order, n);

    return result;
}
```

### Design: Timeout

Timeout requires a timer mechanism. Two options:

**Option A: Timer wheel (dedicated thread)**
A background timer thread manages a hashed + hierarchical timer wheel. `select timeout N` registers a timer callback that wakes the coroutine after N ms.

**Option B: timerfd (Linux)**
For Linux, use `timerfd_create` to create a file descriptor that becomes readable after N ms. The timer FD can be treated as a channel — select just polls it.

**Option C: Per-worker timer check (simplest)**
Each worker, before parking, checks if any timed-out coroutines need waking. Accuracy: ~1ms (acceptable for most use cases).

**Decision:** Option C for v1 (simplest, no extra threads). Option A for v2 if sub-ms timers are needed.

```c
typedef struct jade_timer {
    uint64_t         deadline_ns;  // absolute deadline (clock_gettime MONOTONIC)
    jade_coro_t     *coro;         // coroutine to wake
    struct jade_timer *next;       // linked list
} jade_timer_t;

// Per-scheduler timer list (sorted by deadline)
static jade_timer_t *g_timers = NULL;
static pthread_mutex_t g_timer_lock = PTHREAD_MUTEX_INITIALIZER;

void jade_timer_set(jade_coro_t *c, uint64_t timeout_ms) {
    jade_timer_t *t = malloc(sizeof(jade_timer_t));
    struct timespec now;
    clock_gettime(CLOCK_MONOTONIC, &now);
    t->deadline_ns = now.tv_sec * 1000000000ULL + now.tv_nsec + timeout_ms * 1000000ULL;
    t->coro = c;
    // Insert sorted
    pthread_mutex_lock(&g_timer_lock);
    jade_timer_insert_sorted(t);
    pthread_mutex_unlock(&g_timer_lock);
}

void jade_timer_check(void) {
    struct timespec now;
    clock_gettime(CLOCK_MONOTONIC, &now);
    uint64_t now_ns = now.tv_sec * 1000000000ULL + now.tv_nsec;

    pthread_mutex_lock(&g_timer_lock);
    while (g_timers && g_timers->deadline_ns <= now_ns) {
        jade_timer_t *t = g_timers;
        g_timers = t->next;
        pthread_mutex_unlock(&g_timer_lock);

        t->coro->state = JADE_CORO_READY;
        jade_sched_enqueue(t->coro);
        free(t);

        pthread_mutex_lock(&g_timer_lock);
    }
    pthread_mutex_unlock(&g_timer_lock);
}
```

Workers call `jade_timer_check()` on each iteration of the work loop, before parking.

### Design: Parsing

```rust
// New AST nodes
enum Expr {
    Select(Vec<SelectArm>, Option<Block>, Span),  // arms + optional default block
}

struct SelectArm {
    channel: Expr,          // channel expression
    binding: Option<String>, // variable name for received value (None if send or discard)
    is_send: bool,
    send_value: Option<Expr>, // value to send (if is_send)
    body: Block,
    timeout_ms: Option<Expr>, // if this is a timeout arm
    span: Span,
}
```

Parser: after `select\n INDENT`, parse arms:
- `val from ch_expr\n INDENT body DEDENT` → recv arm
- `from ch_expr\n INDENT body DEDENT` → recv, discard value
- `send ch_expr, val_expr\n INDENT body DEDENT` → send arm
- `timeout expr\n INDENT body DEDENT` → timeout arm
- `default\n INDENT body DEDENT` → default arm (stored separately)

### Design: Codegen

Select compiles to:

1. Allocate `jade_select_case_t cases[N]` on the stack
2. For each arm: evaluate channel expression, fill in case struct
3. `result = call i32 @jade_select(ptr %cases, i32 N, i32 has_default)`
4. `switch i32 %result, label %default [ 0, %arm0  1, %arm1  ... ]`
5. Each arm block binds the received value (if recv) and compiles the body

```rust
ExprKind::Select(arms, default_body) => {
    let n = arms.len();
    let case_ty = /* jade_select_case_t struct type */;
    let cases_alloca = self.entry_alloca(
        case_ty.array_type(n as u32).into(), "select.cases"
    );

    for (i, arm) in arms.iter().enumerate() {
        let ch_val = self.compile_expr(&arm.channel)?;
        let case_ptr = /* GEP into cases_alloca[i] */;
        // Store channel ptr, is_send flag
        // If recv: allocate a buffer for the received value
        // If send: store pointer to send value
    }

    let select_fn = self.module.get_function("jade_select").unwrap();
    let result = self.bld.build_call(select_fn, &[
        cases_alloca.into(),
        i32.const_int(n, false).into(),
        i32.const_int(if default_body.is_some() { 1 } else { 0 }, false).into(),
    ], "select.result").try_as_basic_value().basic().unwrap().into_int_value();

    // Build switch on result
    let default_bb = self.ctx.append_basic_block(fv, "select.default");
    let merge_bb = self.ctx.append_basic_block(fv, "select.merge");
    let arm_bbs: Vec<_> = (0..n).map(|i|
        self.ctx.append_basic_block(fv, &format!("select.arm{i}"))
    ).collect();

    let switch = self.bld.build_switch(result, default_bb,
        &arm_bbs.iter().enumerate().map(|(i, bb)|
            (i32.const_int(i as u64, false), *bb)
        ).collect::<Vec<_>>()
    );

    // Compile each arm
    for (i, (arm, bb)) in arms.iter().zip(arm_bbs.iter()).enumerate() {
        self.bld.position_at_end(*bb);
        if let Some(name) = &arm.binding {
            // Load received value from case buffer
            let val = /* load from cases[i].data */;
            let alloca = self.entry_alloca(/* type */, name);
            self.bld.build_store(alloca, val);
            self.set_var(name, alloca, arm.elem_ty.clone());
        }
        self.compile_block(&arm.body)?;
        if self.no_term() {
            self.bld.build_unconditional_branch(merge_bb);
        }
    }

    // Default arm
    self.bld.position_at_end(default_bb);
    if let Some(body) = default_body {
        self.compile_block(body)?;
    }
    if self.no_term() {
        self.bld.build_unconditional_branch(merge_bb);
    }

    self.bld.position_at_end(merge_bb);
    Ok(void_val)
}
```

### Directory Structure

```

```

### Build Integration

Add a `build.rs` to compile the runtime:

```rust
// build.rs
fn main() {
    let mut build = cc::Build::new();
    build.file("runtime/coro.c")
         .file("runtime/sched.c")
         .file("runtime/deque.c")
         .file("runtime/channel.c")
         .file("runtime/actor.c")
         .file("runtime/select.c")
         .file("runtime/timer.c")
         .flag("-O2")
         .flag("-Wall")
         .flag("-Wextra")
         .flag("-std=c11");

    #[cfg(target_arch = "x86_64")]
    build.file("runtime/context_x86_64.S");

    #[cfg(target_arch = "aarch64")]
    build.file("runtime/context_aarch64.S");

    build.compile("jade_rt");

    println!("cargo:rustc-link-lib=static=jade_rt");
    println!("cargo:rustc-link-lib=pthread");
}
```

The Jade compiler then links `libjade_rt.a` alongside the user's compiled `.o` file. The linker command in `main.rs` adds `-ljade_rt` (or passes the `.a` path directly).

**Alternative (simpler for Jade's current architecture):** Compile the C runtime to `jade_rt.o` separately and pass it via `--link` (Jade already has `--link` for extra objects).

Either approach works. The `build.rs` method is more integrated, the `--link` method is simpler and doesn't couple the Rust build to the C build.

**Decision:** Use `build.rs` + `cc` crate. The runtime is always linked when any concurrency feature is used. When no concurrency features are used (no actors, no channels, no coroutines), the linker's dead-code elimination strips the runtime symbols.

### Linker Changes

In `main.rs`, the linker invocation currently does:
```
cc output.o -o output -lpthread -lm
```

Add the runtime archive:
```
cc output.o jade_rt.a -o output -lpthread -lm
```

The `jade_rt.a` path is determined at Jade compile time. The compiler embeds it via `env!("OUT_DIR")` or a config path.
