## Item 1: Trait Bounds on Generics (P0)

**Goal:** `*max of T: Ord(a: T, b: T) -> T` — reject monomorphization if bound not satisfied.

### Current State

- Generics work via monomorphization (`src/typer/mono.rs`). Type params are `Vec<String>` — bare names, no constraints.
- `ast::Fn.type_params: Vec<String>`, `ast::TypeDef.type_params: Vec<String>`, `ast::EnumDef.type_params: Vec<String>`.
- `Typer.traits: HashMap<String, Vec<TraitMethodSig>>` — trait definitions exist.
- `Typer.trait_impls: HashMap<String, Vec<String>>` — maps type name → implemented trait names.
- Monomorphization: `monomorphize_fn()` in `mono.rs` builds `type_map: HashMap<String, Type>`, calls `substitute_type()`, stamps out specialized function. No bound checking occurs.
- `effective_type_params()` collects implicit type params from untyped parameters.
- `mangle_generic()` builds mangled name from base + concrete types.

### Design

Trait bounds are a **compile-time-only** concern. Zero runtime cost. The check gates monomorphization — if a concrete type doesn't have an `impl Trait for Type`, reject with a clear error. No vtables, no boxing, no indirection.

**Syntax:**
```jade
*max of T: Ord(a: T, b: T) -> T
    if a > b then a else b

*sort of T: Ord(items: Vec of T)
    ...

*serialize of T: Display + Eq(val: T) -> String
    ...
```

Bounds follow the type param after `:`. Multiple bounds separated by `+`. Multiple type params separated by `,` as today.

```rust
fn parse_type_params(&mut self) -> Vec<TypeParam> {
    // Current: after "of", parse comma-separated ident list
    // New: after "of", parse comma-separated TypeParam entries
    //   TypeParam = IDENT [ ':' IDENT { '+' IDENT } ]
    if !self.check(Token::Of) {
        return Vec::new();
    }
    self.advance(); // consume 'of'
    let mut params = Vec::new();
    loop {
        let sp = self.span();
        let name = match self.ident() {
            Ok(n) => n,
            Err(_) => break,
        };
        let bounds = if self.check(Token::Colon) {
            self.advance();
            let mut bs = vec![self.ident().unwrap_or_default()];
            while self.check(Token::Plus) {
                self.advance();
                bs.push(self.ident().unwrap_or_default());
            }
            bs
        } else {
            Vec::new()
        };
        params.push(TypeParam { name, bounds, span: sp });
        if !self.check(Token::Comma) {
            break;
        }
        self.advance();
    }
    params
}
```

```rust
fn type_name_for_bound_check(&self, ty: &Type) -> String {
    match ty {
        Type::Struct(n) => n.clone(),
        Type::I64 => "i64".into(),
        Type::F64 => "f64".into(),
        Type::String => "String".into(),
        Type::Bool => "bool".into(),
        // Primitive types can have trait impls registered
        // via built-in trait resolution (see below)
        _ => format!("{ty}"),
    }
}

fn type_satisfies_trait(&self, type_name: &str, trait_name: &str) -> bool {
    // Check explicit impls
    if let Some(impls) = self.trait_impls.get(type_name) {
        if impls.contains(&trait_name.to_string()) {
            return true;
        }
    }
    // Check built-in trait satisfaction for primitives
    self.builtin_trait_satisfied(type_name, trait_name)
}

fn builtin_trait_satisfied(&self, type_name: &str, trait_name: &str) -> bool {
    match trait_name {
        "Ord" | "Eq" => matches!(
            type_name,
            "i64" | "i32" | "i16" | "i8"
            | "u64" | "u32" | "u16" | "u8"
            | "f64" | "f32" | "String" | "bool"
        ),
        "Display" => true, // all types support Display via log()
        "Add" | "Sub" | "Mul" | "Div" => matches!(
            type_name,
            "i64" | "i32" | "i16" | "i8"
            | "u64" | "u32" | "u16" | "u8"
            | "f64" | "f32"
        ),
        _ => false,
    }
}
```

## Item 2: Iterator Trait (P0)

**Goal:** `trait Iter of T` with `*next() -> Option of T`. Desugar `for x in coll` to `loop { match coll.next() { Some(x) → body, Nothing → break } }`.

### Current State

- `for x in expr` is parsed into `ast::Stmt::For(For)` with `bind`, `iter`, optional `end`/`step`.
- Typer lowering (`src/typer/mod.rs:1353-1393`): infers `bind_ty` from `iter.ty`:
  - `end.is_some() || I64` → numeric range (I64)
  - `Array(et, _)` → element type
  - `Ptr(et)` → inner type
  - Anything else → falls through to `I64` (wrong for user types)
- Codegen (`src/codegen/stmt.rs:314-530`): three specialized paths:
  - `compile_for_array` — GEP with `[0, idx]` into fixed array
  - `compile_for_vec` — GEP into heap data pointer
  - Default — numeric range `i = start; i < end; i += step`
- `Option` is a prelude generic enum: `Some(T)` / `Nothing`. Registered in `register_prelude_types()`.
- Trait system is complete: `TraitDef` parsed, `declare_trait_def` registers sigs, `declare_impl_block` validates methods, vtables generated in codegen.
- Method resolution for `Struct(X)` types already calls `X_method` via `ExprKind::Method`.

### Design — The Carmack Principle: Keep the Fast Path

**Critical constraint:** Array and Vec iteration must remain specialized. These produce tight index loops with GEP — no function call overhead, no Option boxing/unboxing. Adding an Iterator protocol must NOT regress the 0.94× Clang performance.

**Architecture:**
1. Array/Vec/numeric-range `for` loops → **unchanged codegen paths** (fast path)
2. User-defined types implementing `Iter` → **desugar in typer** to loop+match
3. The desugaring produces HIR nodes (Loop + Match on method call), no new codegen needed

This means the Iterator trait is a **typer-level desugaring**, not a codegen concern. The generated HIR feeds into the existing Loop/Match/Method codegen.

**Trait definition** (built into the prelude, like Option/Result):
```jade
trait Iter of T
    *next() -> Option of T
```

**Usage:**
```jade
type Range
    cur: i64
    stop: i64

impl Iter of I64 for Range
    *next self
        if self.cur >= self.stop
            Nothing
        else
            val is self.cur
            self.cur is self.cur + 1
            Some(val)

*main
    for x in Range(cur: 0, stop: 5)
        log x
```

#### Step 6: Desugar `for x in coll` when coll implements Iter

**File:** `src/typer/mod.rs` — in the `ast::Stmt::For(f)` arm of `lower_stmt()`.

After computing `iter` expr and detecting it's not a numeric/array/vec case, check if the type implements `Iter`:

```rust
ast::Stmt::For(f) => {
    let iter = self.lower_expr(&f.iter)?;
    let end = f.end.as_ref().map(|e| self.lower_expr(e)).transpose()?;
    let step = f.step.as_ref().map(|e| self.lower_expr(e)).transpose()?;

    // --- Fast paths: unchanged ---
    if end.is_some() || matches!(iter.ty, Type::I64) {
        // Numeric range: unchanged
        return self.lower_for_numeric(f, iter, end, step, ret_ty);
    }
    match &iter.ty {
        Type::Array(_, _) | Type::Ptr(_) | Type::Vec(_) => {
            // Array/Vec: unchanged fast path
            let bind_ty = match &iter.ty {
                Type::Array(et, _) => *et.clone(),
                Type::Ptr(et) => *et.clone(),
                Type::Vec(et) => *et.clone(),
                _ => unreachable!(),
            };
            return self.lower_for_collection(f, iter, end, step, bind_ty, ret_ty);
        }
        _ => {}
    }

    // --- NEW: Iterator protocol desugaring ---
    if let Type::Struct(ref type_name) = iter.ty {
        if self.type_implements_trait(type_name, "Iter") {
            return self.desugar_for_iter(f, iter, type_name, ret_ty);
        }
    }

    // Fallback: treat as numeric range 0..N
    let bind_ty = Type::I64;
    self.lower_for_collection(f, iter, end, step, bind_ty, ret_ty)
}
```

#### Step 7: The desugaring function

**File:** `src/typer/mod.rs`

```rust
fn type_implements_trait(&self, type_name: &str, trait_name: &str) -> bool {
    self.trait_impls
        .get(type_name)
        .map(|impls| impls.contains(&trait_name.to_string()))
        .unwrap_or(false)
}

fn iter_element_type(&mut self, type_name: &str) -> Type {
    // Look up (type_name, "Iter") in trait_impl_type_args
    if let Some(args) = self.trait_impl_type_args.get(&(type_name.into(), "Iter".into())) {
        if let Some(t) = args.first() {
            return t.clone();
        }
    }
    // Fallback: look at next() return type
    let fn_name = format!("{type_name}_next");
    if let Some((_, _, ret)) = self.fns.get(&fn_name) {
        // ret is Enum("Option_X") — extract X
        if let Type::Enum(ref ename) = ret {
            if let Some(stripped) = ename.strip_prefix("Option_") {
                return self.parse_type_from_name(stripped);
            }
        }
    }
    Type::I64
}

fn desugar_for_iter(
    &mut self,
    f: &ast::For,
    iter_expr: hir::Expr,
    type_name: &str,
    ret_ty: &Type,
) -> Result<hir::Stmt, String> {
    let elem_ty = self.iter_element_type(type_name);
    let span = f.span;

    // 1. Bind the iterator to a mutable variable
    let iter_bind_id = self.fresh_id();
    let iter_var_name = format!("__iter_{}", f.bind);

    // 2. Build: __iter.next() call
    let method_name = format!("{type_name}_next");
    let (fn_id, _, ret) = self.fns.get(&method_name).cloned()
        .ok_or_else(|| format!("{type_name} implements Iter but has no next() method"))?;
    let iter_var_expr = hir::Expr {
        kind: hir::ExprKind::Var(iter_bind_id, iter_var_name.clone()),
        ty: iter_expr.ty.clone(),
        span,
    };
    let next_call = hir::Expr {
        kind: hir::ExprKind::Method(
            Box::new(iter_var_expr.clone()),
            method_name,
            "next".into(),
            vec![],
        ),
        ty: ret,
        span,
    };

    // 3. Build match arms:
    //    Some(x) → { body }
    //    Nothing → break
    let bind_id = self.fresh_id();
    let some_pat = hir::Pat::Ctor(
        "Some".into(), 0,
        vec![hir::Pat::Bind(bind_id, f.bind.clone(), elem_ty.clone(), span)],
        span,
    );
    let nothing_pat = hir::Pat::Ctor("Nothing".into(), 1, vec![], span);

    // Lower the loop body in a scope with the bound variable
    self.push_scope();
    self.define_var(&f.bind, VarInfo {
        def_id: bind_id,
        ty: elem_ty.clone(),
        ownership: Ownership::Owned,
    });
    let body = self.lower_block_no_scope(&f.body, ret_ty)?;
    self.pop_scope();

    let some_arm = hir::Arm {
        pat: some_pat,
        guard: None,
        body,
        span,
    };
    let nothing_arm = hir::Arm {
        pat: nothing_pat,
        guard: None,
        body: vec![hir::Stmt::Break(None, span)],
        span,
    };

    let match_stmt = hir::Stmt::Match(hir::Match {
        subject: next_call,
        arms: vec![some_arm, nothing_arm],
        ty: Type::Void,
        span,
    });

    // 4. Wrap in Loop { match iter.next() ... }
    let loop_stmt = hir::Stmt::Loop(hir::Loop {
        body: vec![match_stmt],
        span,
    });

    // 5. Wrap in block: { let __iter = expr; loop { ... } }
    // Actually, since Jade doesn't have block-as-stmt directly,
    // we emit a Bind + Loop at the current scope level.
    // The caller is lower_stmt() which returns a single Stmt.
    // Solution: wrap in a synthetic block via Expr::Block.

    // Simpler: emit the iter binding in the enclosing scope and return the loop.
    // The typer's scope management handles this naturally.

    // First define the iterator variable
    self.define_var(&iter_var_name, VarInfo {
        def_id: iter_bind_id,
        ty: iter_expr.ty.clone(),
        ownership: Ownership::Owned,
    });

    // Emit both stmts as a synthetic block
    // Since lower_stmt returns one Stmt, we need to package this.
    // Use Stmt::Expr(ExprKind::Block(...)):
    let bind_stmt = hir::Stmt::Bind(hir::Bind {
        def_id: iter_bind_id,
        name: iter_var_name,
        value: iter_expr,
        ty: Type::Struct(type_name.to_string()),
        ownership: Ownership::Owned,
        span,
    });

    Ok(hir::Stmt::Expr(hir::Expr {
        kind: hir::ExprKind::Block(vec![bind_stmt, loop_stmt]),
        ty: Type::Void,
        span,
    }))
}
```

**Key insight:** The desugaring produces `{ let __iter = expr; loop { match __iter.next() { Some(x) → body, Nothing → break } } }`. This is pure HIR — no new codegen paths needed. The existing Loop, Match, Method, and Break codegen handle it all.

#### Step 9: Handle mutable iterator state

The desugared loop calls `.next()` on the iterator, which typically mutates internal state (e.g., advancing a cursor). The iterator is bound to a local variable via `Bind`, and `.next()` modifies it through `self`.

**Method call semantics:** In Jade, methods that take `self` receive the struct by value (copied in). For mutation to work, the method needs to operate on a pointer to the struct. Check how existing mutable methods work:

In `compile_method()` (`src/codegen/expr.rs`), the receiver is compiled, and if it's a struct, the value is passed. For mutation to persist, the method must take self by reference (pointer).

**Resolution:** The iterator variable `__iter` must be stack-allocated (which `Bind` does via `entry_alloca`). The `.next()` call passes a pointer to this alloca. This is the existing behavior for struct method calls when the receiver is a variable — the codegen loads the variable's pointer and passes it. Verify this is the case.

If the current method call compiles by-value (copies self into the call), mutation won't persist. In that case, the `.next()` call needs to be modified to pass the receiver's alloca pointer directly. This is the same issue actor handlers solve by binding state fields via GEP.

**Verification task:** Read `compile_method()` in `expr.rs` to confirm struct method receiver semantics. If by-value, this is a pre-existing design constraint. The fix is either:
- (a) Pass methods a pointer to the receiver when the method mutates `self`, or
- (b) The `.next()` method returns `(Option of T, Self)` — tuple of result + updated state. The desugaring rebinds `__iter` on each call.

Option (b) is cleaner and functional — no hidden mutation, works with Perceus. The desugaring becomes:

```
loop {
    result, __iter is __iter.next()  // next returns (Option of T, Self)
    match result { ... }
}
```

**However**, checking the method compilation in `expr.rs` — the receiver is compiled as a value, not a pointer. So approach (a) requires changing method call semantics globally (too invasive). Approach (b) requires changing the Iter trait signature.

**Best approach for Jade:** Actually, Jade methods already receive self by value and the receiver is looked up as a variable. The codegen does:


## Item 3: Associated Types (P1)

**Goal:** `trait Iter with type Item`. Needed for map/filter/zip chains.

### Current State

- Traits have `methods: Vec<TraitMethodSig>` — no associated types.
- Trait type params use `type_params: Vec<String>` (after Item 1: `Vec<TypeParam>`).
- MonomorphizaTion does type substitution via `substitute_type()` with `HashMap<String, Type>`.
- Method return types are resolved from the concrete impl's function signature.

### Design

Associated types let a trait define named type members that impls specify concretely:

```jade
trait Iter
    type Item
    *next() -> Option of Item

impl Iter for IntRange
    type Item is I64
    *next self -> Option of I64
        ...

trait Collection
    type Item
    type Iter: Iter
    *iter() -> Iter
```

This is more expressive than raw type parameters: the type is determined by the implementor, not the caller. A single struct implements `Iter` with a fixed `Item` — callers don't specify `T`.

**Key difference from type params:** `Iter of T` requires the caller to know `T`. `Iter with type Item` lets the impl choose `Item`. For iterator chains (`map`, `filter`), the output `Item` type changes — associated types make this natural.

## Item 4: String Iteration (P2)

**Goal:** `for ch in str` iterates Unicode codepoints. Needs UTF-8 decoder.

### Current State

- Strings are SSO: tag byte at offset 23 discriminates inline (≤23 bytes, bit7=1) vs heap ({ptr,len,cap}, bit7=0).
- `string_len()` returns byte length. `string_data()` returns pointer to byte data.
- `string_char_at()` exists — returns byte value at index (`i8_type` load).
- No UTF-8 awareness anywhere. `char_at` returns raw bytes.
- For-loops: the typer handles Array/Vec/Ptr/I64. String is not handled.

### Design

Iterating codepoints requires a UTF-8 state machine. UTF-8 encoding:

| Byte pattern | Codepoint range | Sequence length |
|---|---|---|
| `0xxxxxxx` | U+0000–U+007F | 1 |
| `110xxxxx 10xxxxxx` | U+0080–U+07FF | 2 |
| `1110xxxx 10xxxxxx 10xxxxxx` | U+0800–U+FFFF | 3 |
| `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` | U+10000–U+10FFFF | 4 |

**Element type:** `i64` (Unicode scalar value, 0–0x10FFFF). Not `char` — Jade has no char type. The codepoint is an integer.

**Performance strategy:** Emit the UTF-8 decoder as inline LLVM IR. No function call per codepoint. The decoder is a chain of comparisons on the leading byte with direct bit extraction — zero branches for ASCII (the common case), 2-3 branches for multi-byte.

**Two implementation options:**

1. **Desugar to Iterator protocol:** Create a built-in `StringIter` type with `Iter of I64` impl. This reuses the iterator infrastructure from Item 2. Clean but has the overhead of the iterator protocol (match on Option per character).

2. **Specialized codegen path:** Add `compile_for_string()` to `stmt.rs`, like `compile_for_array` and `compile_for_vec`. Direct index loop with inline UTF-8 decode. Maximum performance.

**Decision: Specialized codegen path.** String iteration is hot-path code. The overhead of constructing/destructing `Option_i64` per character, plus the match dispatch, is unnecessary when we can emit a tight byte-scanning loop. The specialized path produces the same quality code as hand-written C.

ASCII fast-path: If the leading byte has bit 7 clear, the codepoint IS the byte. This path should be a single comparison + branch, falling through to the multi-byte decoder only when needed. ~95% of real-world text is ASCII — this optimization matters.

**File:** `src/codegen/stmt.rs`

```rust
fn compile_for_string(
    &mut self,
    f: &hir::For,
) -> Result<Option<BasicValueEnum<'ctx>>, String> {
    let fv = self.cur_fn.unwrap();
    let i8t = self.ctx.i8_type();
    let i32t = self.ctx.i32_type();
    let i64t = self.ctx.i64_type();

    // Get string data pointer and byte length
    let str_val = self.compile_expr(&f.iter)?;
    let data_ptr = self.string_data(str_val)?.into_pointer_value();
    let byte_len = self.string_len(str_val)?.into_int_value();

    // Allocate byte index and codepoint variable
    let idx_alloca = self.entry_alloca(i64t.into(), "__str_idx");
    b!(self.bld.build_store(idx_alloca, i64t.const_int(0, false)));
    let cp_alloca = self.entry_alloca(i64t.into(), &f.bind);
    self.set_var(&f.bind, cp_alloca, Type::I64);

    // Basic blocks
    let cond_bb = self.ctx.append_basic_block(fv, "str.cond");
    let decode_bb = self.ctx.append_basic_block(fv, "str.decode");
    let body_bb = self.ctx.append_basic_block(fv, "str.body");
    let inc_bb = self.ctx.append_basic_block(fv, "str.inc");
    let end_bb = self.ctx.append_basic_block(fv, "str.end");

    b!(self.bld.build_unconditional_branch(cond_bb));

    // --- Condition: idx < byte_len ---
    self.bld.position_at_end(cond_bb);
    let idx = b!(self.bld.build_load(i64t, idx_alloca, "idx")).into_int_value();
    let cmp = b!(self.bld.build_int_compare(IntPredicate::ULT, idx, byte_len, "str.cmp"));
    b!(self.bld.build_conditional_branch(cmp, decode_bb, end_bb));

    // --- Decode: read leading byte, determine sequence length, extract codepoint ---
    self.bld.position_at_end(decode_bb);
    let idx = b!(self.bld.build_load(i64t, idx_alloca, "idx")).into_int_value();

    // Load leading byte
    let byte_ptr = unsafe {
        b!(self.bld.build_gep(i8t, data_ptr, &[idx], "str.bp"))
    };
    let b0 = b!(self.bld.build_load(i8t, byte_ptr, "b0")).into_int_value();
    let b0_32 = b!(self.bld.build_int_z_extend(b0, i32t, "b0.32"));

    // Determine sequence length and extract codepoint
    // Strategy: chain of comparisons, building codepoint progressively.
    //
    // ASCII: b0 < 0x80 → cp = b0, len = 1
    // 2-byte: b0 & 0xE0 == 0xC0 → cp = (b0&0x1F)<<6 | (b1&0x3F), len = 2
    // 3-byte: b0 & 0xF0 == 0xE0 → cp = (b0&0x0F)<<12 | (b1&0x3F)<<6 | (b2&0x3F), len = 3
    // 4-byte: b0 & 0xF8 == 0xF0 → cp = (b0&0x07)<<18 | (b1&0x3F)<<12 | (b2&0x3F)<<6 | (b3&0x3F), len = 4

    let ascii_bb = self.ctx.append_basic_block(fv, "str.ascii");
    let mb2_test_bb = self.ctx.append_basic_block(fv, "str.mb2t");
    let mb2_bb = self.ctx.append_basic_block(fv, "str.mb2");
    let mb3_test_bb = self.ctx.append_basic_block(fv, "str.mb3t");
    let mb3_bb = self.ctx.append_basic_block(fv, "str.mb3");
    let mb4_bb = self.ctx.append_basic_block(fv, "str.mb4");
    let decoded_bb = self.ctx.append_basic_block(fv, "str.decoded");

    // Test ASCII: b0 < 128
    let is_ascii = b!(self.bld.build_int_compare(
        IntPredicate::ULT, b0_32,
        i32t.const_int(0x80, false), "is_ascii"
    ));
    b!(self.bld.build_conditional_branch(is_ascii, ascii_bb, mb2_test_bb));

    // --- ASCII path ---
    self.bld.position_at_end(ascii_bb);
    let ascii_cp = b!(self.bld.build_int_z_extend(b0_32, i64t, "ascii.cp"));
    let ascii_adv = i64t.const_int(1, false);
    b!(self.bld.build_unconditional_branch(decoded_bb));

    // --- 2-byte test: (b0 & 0xE0) == 0xC0 ---
    self.bld.position_at_end(mb2_test_bb);
    let m2 = b!(self.bld.build_and(b0_32, i32t.const_int(0xE0, false), "m2"));
    let is_2b = b!(self.bld.build_int_compare(
        IntPredicate::EQ, m2, i32t.const_int(0xC0, false), "is_2b"
    ));
    b!(self.bld.build_conditional_branch(is_2b, mb2_bb, mb3_test_bb));

    // --- 2-byte decode ---
    self.bld.position_at_end(mb2_bb);
    let idx_1 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(1, false), "i1"));
    let bp1 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_1], "bp1")) };
    let b1 = b!(self.bld.build_load(i8t, bp1, "b1")).into_int_value();
    let b1_32 = b!(self.bld.build_int_z_extend(b1, i32t, "b1.32"));
    // cp = ((b0 & 0x1F) << 6) | (b1 & 0x3F)
    let hi = b!(self.bld.build_and(b0_32, i32t.const_int(0x1F, false), "hi2"));
    let hi_s = b!(self.bld.build_left_shift(hi, i32t.const_int(6, false), "hi2s"));
    let lo = b!(self.bld.build_and(b1_32, i32t.const_int(0x3F, false), "lo2"));
    let cp2_32 = b!(self.bld.build_or(hi_s, lo, "cp2"));
    let cp2 = b!(self.bld.build_int_z_extend(cp2_32, i64t, "cp2.64"));
    let adv2 = i64t.const_int(2, false);
    b!(self.bld.build_unconditional_branch(decoded_bb));

    // --- 3-byte test: (b0 & 0xF0) == 0xE0 ---
    self.bld.position_at_end(mb3_test_bb);
    let m3 = b!(self.bld.build_and(b0_32, i32t.const_int(0xF0, false), "m3"));
    let is_3b = b!(self.bld.build_int_compare(
        IntPredicate::EQ, m3, i32t.const_int(0xE0, false), "is_3b"
    ));
    b!(self.bld.build_conditional_branch(is_3b, mb3_bb, mb4_bb));

    // --- 3-byte decode ---
    self.bld.position_at_end(mb3_bb);
    let idx_1 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(1, false), "i1"));
    let idx_2 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(2, false), "i2"));
    let bp1 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_1], "bp1")) };
    let bp2 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_2], "bp2")) };
    let b1 = b!(self.bld.build_load(i8t, bp1, "b1")).into_int_value();
    let b2 = b!(self.bld.build_load(i8t, bp2, "b2")).into_int_value();
    let b1_32 = b!(self.bld.build_int_z_extend(b1, i32t, "b1.32"));
    let b2_32 = b!(self.bld.build_int_z_extend(b2, i32t, "b2.32"));
    // cp = ((b0&0x0F)<<12) | ((b1&0x3F)<<6) | (b2&0x3F)
    let h3 = b!(self.bld.build_and(b0_32, i32t.const_int(0x0F, false), "h3"));
    let h3s = b!(self.bld.build_left_shift(h3, i32t.const_int(12, false), "h3s"));
    let m3b = b!(self.bld.build_and(b1_32, i32t.const_int(0x3F, false), "m3b"));
    let m3s = b!(self.bld.build_left_shift(m3b, i32t.const_int(6, false), "m3s"));
    let l3 = b!(self.bld.build_and(b2_32, i32t.const_int(0x3F, false), "l3"));
    let cp3_a = b!(self.bld.build_or(h3s, m3s, "cp3a"));
    let cp3_32 = b!(self.bld.build_or(cp3_a, l3, "cp3"));
    let cp3 = b!(self.bld.build_int_z_extend(cp3_32, i64t, "cp3.64"));
    let adv3 = i64t.const_int(3, false);
    b!(self.bld.build_unconditional_branch(decoded_bb));

    // --- 4-byte decode (fallthrough from 3-byte test) ---
    self.bld.position_at_end(mb4_bb);
    let idx_1 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(1, false), "i1"));
    let idx_2 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(2, false), "i2"));
    let idx_3 = b!(self.bld.build_int_nuw_add(idx, i64t.const_int(3, false), "i3"));
    let bp1 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_1], "bp1")) };
    let bp2 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_2], "bp2")) };
    let bp3 = unsafe { b!(self.bld.build_gep(i8t, data_ptr, &[idx_3], "bp3")) };
    let b1 = b!(self.bld.build_load(i8t, bp1, "b1")).into_int_value();
    let b2 = b!(self.bld.build_load(i8t, bp2, "b2")).into_int_value();
    let b3 = b!(self.bld.build_load(i8t, bp3, "b3")).into_int_value();
    let b1_32 = b!(self.bld.build_int_z_extend(b1, i32t, "b1.32"));
    let b2_32 = b!(self.bld.build_int_z_extend(b2, i32t, "b2.32"));
    let b3_32 = b!(self.bld.build_int_z_extend(b3, i32t, "b3.32"));
    // cp = ((b0&0x07)<<18) | ((b1&0x3F)<<12) | ((b2&0x3F)<<6) | (b3&0x3F)
    let h4 = b!(self.bld.build_and(b0_32, i32t.const_int(0x07, false), "h4"));
    let h4s = b!(self.bld.build_left_shift(h4, i32t.const_int(18, false), "h4s"));
    let m4a = b!(self.bld.build_and(b1_32, i32t.const_int(0x3F, false), "m4a"));
    let m4as = b!(self.bld.build_left_shift(m4a, i32t.const_int(12, false), "m4as"));
    let m4b = b!(self.bld.build_and(b2_32, i32t.const_int(0x3F, false), "m4b"));
    let m4bs = b!(self.bld.build_left_shift(m4b, i32t.const_int(6, false), "m4bs"));
    let l4 = b!(self.bld.build_and(b3_32, i32t.const_int(0x3F, false), "l4"));
    let cp4_a = b!(self.bld.build_or(h4s, m4as, "cp4a"));
    let cp4_b = b!(self.bld.build_or(cp4_a, m4bs, "cp4b"));
    let cp4_32 = b!(self.bld.build_or(cp4_b, l4, "cp4"));
    let cp4 = b!(self.bld.build_int_z_extend(cp4_32, i64t, "cp4.64"));
    let adv4 = i64t.const_int(4, false);
    b!(self.bld.build_unconditional_branch(decoded_bb));

    // --- Merge: phi nodes for codepoint and advance ---
    self.bld.position_at_end(decoded_bb);
    let cp_phi = b!(self.bld.build_phi(i64t, "cp"));
    cp_phi.add_incoming(&[
        (&ascii_cp, ascii_bb),
        (&cp2, mb2_bb),
        (&cp3, mb3_bb),
        (&cp4, mb4_bb),
    ]);
    let adv_phi = b!(self.bld.build_phi(i64t, "adv"));
    adv_phi.add_incoming(&[
        (&ascii_adv, ascii_bb),
        (&adv2, mb2_bb),
        (&adv3, mb3_bb),
        (&adv4, mb4_bb),
    ]);

    // Store codepoint and advance index
    b!(self.bld.build_store(cp_alloca, cp_phi.as_basic_value()));
    let cur_idx = b!(self.bld.build_load(i64t, idx_alloca, "ci")).into_int_value();
    let new_idx = b!(self.bld.build_int_nuw_add(
        cur_idx, adv_phi.as_basic_value().into_int_value(), "ni"
    ));
    b!(self.bld.build_store(idx_alloca, new_idx));

    // Fall through to body
    b!(self.bld.build_unconditional_branch(body_bb));

    // --- Body ---
    self.bld.position_at_end(body_bb);
    self.loop_stack.push(super::LoopCtx {
        continue_bb: cond_bb,
        break_bb: end_bb,
    });
    self.compile_block(&f.body)?;
    self.loop_stack.pop();
    if self.no_term() {
        b!(self.bld.build_unconditional_branch(cond_bb));
    }

    self.bld.position_at_end(end_bb);
    Ok(None)
}
```

This produces the same quality code as a hand-optimized C UTF-8 decoder:
- ASCII fast path: 1 comparison + branch (no masking, no shifts)
- Multi-byte: exactly the minimum operations needed
- No function calls, no heap allocation, no Option boxing
- LLVM can further optimize the phi chains and branches

#### Step 4: SSO-aware data pointer

The `string_data()` function already handles SSO — it returns the correct pointer regardless of inline vs heap. Same for `string_len()`. So the decoder works transparently with both SSO inline strings and heap strings.

## Performance Impact

- **Trait bounds:** Zero runtime cost (compile-time check only).
- **Iterator trait:** Zero cost for Array/Vec/numeric (unchanged fast paths). User iterator types: one function call + match per element. Competitive with C for-each on linked structures.
- **Associated types:** Zero runtime cost (compile-time resolution).
- **String iteration:** Inline UTF-8 decode. ASCII fast path is ~2 instructions. Comparable to `mbrtowc()` in C but without the function call overhead.

## Risks and Mitigations

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Method mutation for iterators (by-value self) | High | Addressed via pointer-based self for Iter methods; `ExprKind::IterNext` passes alloca pointer |
| Option monomorphization mismatch in desugaring | Medium | Explicit `monomorphize_enum("Option", ...)` in `desugar_for_iter` |
| TypeParam threading breaks existing generics | Medium | Backward compatible: `TypeParam { name, bounds: vec![] }` equals current `String` |
| SSO string pointer invalidation during iteration | Low | `string_data()` returns stable pointer; SSO inline data IS the struct, stack-allocated, doesn't move |
| Trait bound checking on primitive types | Low | Hardcoded `builtin_trait_satisfied()` table for numeric/string/bool types |
