道

Principles

一

Complete Type Inference

Static typing without annotations. The compiler infers types automatically.

二

Optimal Managed Memory

No Garbage Collector. Allocations and frees are handled intelligently.

三

Readable Syntax

No arcane syntactical invocations or symbolic litter. Jinn reads like pseudocode.

四

Native Performance

Heavily optimized native binaries run as fast as C. Sometimes faster.

五

Best of Both Worlds

Optimal developer efficiency and code readability combined with native performance.

六

Batteries Included

Persistent object storage, native actor framework, comprehensive standard library.

筆

Code

An intuitive blend of object-oriented and functional styles.

Functions Patterns & Pipelines 
# Pattern-directed functions
*fib(0) is 0
*fib(1) is 1
*fib(n) is fib(n - 1) + fib(n - 2)

# Higher-order functions
*apply(f, x) is f(x)
*double(x) is x * 2
*add_one(x) is x + 1

# Pipelines chain naturally
result is 5 ~ double ~ add_one
log(result)  # 11
Type System Structs & Enums 
type Vec3
    x as f64
    y as f64
    z as f64

*dot(a, b)
    a.x * b.x + a.y * b.y + a.z * b.z

enum Op
    Add(i64, i64)
    Mul(i64, i64)
    Neg(i64)

*eval(op as Op)
    match op
        Add(a, b) ? a + b
        Mul(a, b) ? a * b
        Neg(a)    ? 0 - a
Concurrency Actors & Channels 
actor Counter
    count is 0

    @ping n
        count is count + n

*main
    w is spawn Counter
    w.ping(1)
    w.ping(5)

    # Typed buffered channels
    ch is channel of i64(1024)
    send ch, 42
    val is receive ch
    log(val)  # 42
Persistence Built-in Stores 
store records
    key
    value

*main
    i is 0
    while i < 100
        insert records i, i * 7
        i is i + 1

    # Compile-time validated queries
    r is records where key equals 42
    log(r.value)  # 294

    c is count records
    log(c)  # 100
器

Features

Perceus Reference Counting

Compile-time optimized memory management. Statically inserted drops, no GC pauses.

Type Inference

Hindley-Milner + bidirectional inference. Write code, not annotations.

Pattern Matching

Exhaustive match with destructuring, guards, nested patterns, and or-patterns.

Actors & Channels

Lightweight actors on a work-stealing scheduler. Typed channels. Message-passing with send.

C FFI

Direct extern declarations. Automatic header binding with jinn bind.

Persistent Stores

Typed, file-backed data stores with compile-time validated queries and transactions.

Generics & Traits

Monomorphized generics with of syntax. Trait-based polymorphism with dyn dispatch.

Coroutines & Generators

Cooperative context switching with yield. Lazy sequences that produce values on demand.

Error Values

Errors are values, not exceptions. Define with err, return with !, handle with match.

Syntax

Bindings & Control Flow
x is 42
name is 'jinn'
result is x > 0 ? 'yes' ! 'no'

if x > 10
    log('big')
unless x equals 0
    log('nonzero')
Functions & Lambdas
*add(a, b) is a + b
*double(x) is x * 2
*apply(f, x) is f(x)

# Lambda syntax
square is |x| x * x

# Pipeline operator
5 ~ double ~ add_one
Types & Generics
type Point
    x as f64
    y as f64

*max of T(a as T, b as T)
    a > b ? a ! b

for i from 0 to 100
    log(i)
Enums & Matching
enum Shape
    Circle(f64)
    Rect(f64, f64)

*area(s as Shape)
    match s
        Circle(r) ? 3.14159 * r * r
        Rect(w, h) ? w * h
Actors & Messages
actor Worker
    count
    @ping n
        count is count + n

c is spawn Worker
send c, @ping(5)
Stores & Queries
store users
    name
    age

insert users 'Alice', 30
r is users where name equals 'Alice'
log(r.age)  # 30

Get Started

Build the Compiler

# Requires Rust and LLVM 21
$ git clone https://github.com/example/jinn
$ cd jinn
$ cargo build --release

Write Your First Program

# hello.jn
*greet(name)
    'hello, {name}'

*main
    log greet('world')

Compile & Run

$ jinnc hello.jn -o hello
$ ./hello
hello, world
