Jade Language Guide

Jade is a compiled systems language focused on readability, static typing with inference, actor concurrency, and built-in persistence.

1. Quick Start

jade init demo
cd demo
jade run
jade test
jade build -o demo_bin

If your environment uses the legacy binary name, replace jade with jadec.

2. Project Layout and Modules

demo/
  project.jade
  source/
    main.jade
    analytics.jade
    workers/
      ingest.jade

Imports use slash paths:

use analytics
use workers/ingest
use std/time

Call module functions with method-style qualification:

sum is analytics.sum_range(1, 100)
value is ingest.transform(42)
now is time.monotonic()

3. Compiler and CLI Workflows

  • jade build: compile project entry.
  • jade run: compile and execute.
  • jade test: run inline tests.
  • jade check: type-check only.
  • jade fmt: format source files.
  • jade bind header.h: generate extern declarations from C headers.
  • jade fetch: resolve and cache package dependencies.
  • jade update: refresh lockfile dependency commits.
  • jade package: emit jade.pkg and source archive.
  • jade publish: publish by creating git tag vX.Y.Z.

4. Package Includes and Publish Workflow

Dependencies are declared in project.jade and resolved through git URLs + semantic version tags.

name is 'consumer'
version is '0.1.0'
entry is 'source/main.jade'
require 'mathlib' 'https://example.com/mathlib.git' '0.2.1'

Typical flow:

# package author
jade package
jade publish --push

# package consumer
jade fetch
jade build
jade run

For local development-only dependency tests, you can opt in to non-HTTPS URLs by setting JADE_ALLOW_NON_HTTPS_DEPS=1 and using file:// URLs.

5. Actors and Loop Handlers

Actor handlers are declared inside actor blocks; messages are sent with regular method-call syntax.

actor Counter
    total as i64
    *add n as i64
        total is total + n
    *show
        log(total)

*main
    c is spawn Counter
    c.add(10)
    c.add(20)
    c.show()

Loop handlers are supported with optional sleep in milliseconds:

actor Ticker
    ticks as i64
    *loop 100
        ticks is ticks + 1

6. Persistent Store

store users
    name as String
    age as i64

*main
    insert users 'Ada', 36
    insert users 'Bob', 24
    set users where name equals 'Bob' age 25
    row is users where name equals 'Bob'
    log(row.age)
    log(count users)

Store records persist to disk and are suitable for fast embedded persistence without an external database server.

7. Testing

*add(a, b)
    a + b

test 'addition'
    assert add(2, 3) equals 5

*main
    log('normal runtime entry')

Run all inline tests with jade test.

8. Cross Compilation and Standalone Builds

jade build --target aarch64-unknown-linux-gnu -o app-aarch64
jade build --target riscv64-unknown-linux-gnu --cpu rocket --features +m,+a
jade build --target wasm32-wasi --standalone -o app.wasm

--standalone enables no-runtime/no-libc-style builds useful for embedded, boot, and low-level environments where minimal linkage is required.

9. Performance Notes

  • Use --opt 3 for release builds.
  • Use --lto for cross-module optimization.
  • Use benchmark subsets to track actor/store throughput and latency over time.
  • Use --deterministic-fp for reproducible floating-point behavior.

10. Alpha Release Checklist

  • Multi-file project build and module import pass.
  • Standard library imports validated in real projects.
  • Package include + lock + fetch flow pass.
  • Package publish flow (git tag + optional push) pass.
  • Actor messaging and loop behavior pass.
  • Persistent store insert/query/update/count pass.
  • Cross-target object/link workflows validated for target matrix.
  • Smoke benchmark baseline captured for actor/store paths.
